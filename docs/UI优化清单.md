# UI 优化清单

> 面向对象：儿童（6–9 岁）主用户 + 家长 / 老师 / 临床医生次要用户
> 场景：非营利，医院 / 学校 / 家庭
> 目标：在当前"温暖儿童友好"底子上，把质感从"能用"提升到"专业且克制的儿童医疗工具"
>
> 参考风格：Headspace Kids（克制的插画 + 柔和渐变 + 呼吸感留白），Duolingo（强视觉反馈但不喧闹的配色 + 清晰 CTA），Linear（极致的 token 化 + 阴影纪律），Notion（内容优先的排版）。
>
> 本清单筛选了前 ~25 个最有性价比的改进点。每一条含：**现状 / 问题 / 建议 / 优先级**。

---

## 1. 设计系统层面 (P0 - 基础)

### 1.1 缺少中性灰阶（neutral scale）
- **现状**：`app.css` 只有一个 `--text-muted: #8B6F5C`（暖棕）和硬编码的 `#E8DDD4`、`#F0E8E0`、`#F5F0EB`、`#FFE8D1`、`#FFE0CC` 等 5+ 个散落的浅棕色。
- **问题**：边框、分割线、禁用态、背景底色没有命名 token，改配色时要改几十处；而且这几个值彼此饱和度不统一，看起来"脏"而不是"暖"。
- **建议**：新增 `--neutral-50` → `--neutral-900` 9 阶梯（基于暖白 `#FFF8F0` 向 `#2B1810` 插值，可用 HSL 保持同色相 H≈28），把所有 border / divider / skeleton / disabled bg 全部替换。参考 Radix `sand` 或 `warm-gray` 的阶梯命名。
- **优先级**：P0

### 1.2 字号阶梯太粗、缺少 caption 与超大标题
- **现状**：`--font-size-sm(14) / base(16) / lg(20) / xl(24) / 2xl(32) / 3xl(40)`。
- **问题**：(1) 缺少 12px caption（大量 stat-label 硬编码为 `font-size: 12px`）；(2) 儿童用户需要比成人更大的正文，主交互应该有 18–22 的"儿童正文"档；(3) 从 24 跳 32 再跳 40 变化率不均匀。
- **建议**：改为 `--text-xs: 12 / --text-sm: 14 / --text-base: 16 / --text-kid: 18 / --text-lg: 22 / --text-xl: 28 / --text-2xl: 36 / --text-3xl: 48 / --text-hero: 64`（模数约 1.25–1.33）。并引入 `--font-weight-regular/medium/semibold/bold/black` token，避免 `600/700/800` 散落。
- **优先级**：P0

### 1.3 阴影与圆角层级偏扁平
- **现状**：shadow 全部是垂直 offset + 暖棕色 rgba，没有方向感；radius 只有 12/20/28/pill。
- **问题**：所有卡片看起来像同一张纸贴在同一张桌子上，缺少层级；没有 `--radius-xs: 6` 用于小芯片、badge，也没有 `--radius-xl: 32` 用于 hero。
- **建议**：
  - shadow 改为双层：`--shadow-sm: 0 1px 2px rgba(43,24,16,.04), 0 1px 1px rgba(43,24,16,.03)`；`--shadow-md` 加入轻微彩色层（如 `0 8px 24px -4px rgba(255,140,66,.08)`）让暖色调顺延到阴影；浮层 `--shadow-pop` 用于 hover。
  - 新增 `--radius-xs: 6` 与 `--radius-xl: 32`。
- **优先级**：P0

### 1.4 颜色语义层缺失（state colors）
- **现状**：`--success = --secondary`（都是 #4ECDC4），`--warning = --tertiary`（#FFD166），直接复用品牌色。`risk_level` 用 `secondary / tertiary-dark / error` 临时拼。
- **问题**：品牌色和状态色混用会让"低风险=品牌色"这种信息暗示反过来污染品牌（CTA 也用同色时，用户看见 CTA 会误读为"安全"而不是"行动"）。也没有 info 蓝。
- **建议**：
  - 拆成品牌色 `--brand-primary/secondary/tertiary` + 独立语义色 `--state-success / --state-warning / --state-danger / --state-info`，语义色整体去饱和 10–15% 以与品牌区分。
  - `risk_level` 建议专用：低 `#3BABA4`（青绿去饱和）/ 中 `#F4A261`（琥珀）/ 高 `#C94444`（收敛的砖红，不要纯红），三档都偏"大地色"，更医疗、少惊吓。
- **优先级**：P0

### 1.5 硬编码颜色值散落在组件里
- **现状**：`CalibrationCanvas.svelte` 内硬编码 `#37302B / #FF8C42 / #4ECDC4 / #FFD166 / #FFF8F0 / #4A3810` 等 10+ 处；`screening/running` 和 `result` 都反复写 `#E8DDD4`、`#F0E0CC`、`#FFE0CC`。
- **问题**：Canvas 绘制拿不到 CSS 变量是事实，但这些值应该从一个 JS 常量文件导出，而不是各写各的。
- **建议**：新建 `src/lib/design/tokens.js` 导出所有 Canvas 能用到的色值常量（来源与 CSS 变量同构），在 Svelte 组件顶部引用；同时在 `app.css` 设置一份 `:root --canvas-bg` 等镜像变量，保证单一真源。
- **优先级**：P0

### 1.6 缺少 motion token
- **现状**：`transition` 里硬编码 `0.15s / 0.2s / 0.5s / 0.8s / 1s` 各自为政；`ease / ease-out / linear` 混用。
- **问题**：动效节奏不一致，儿童 UI 特别容易感到"这里快那里慢"很跳脱。
- **建议**：token 化 `--ease-out: cubic-bezier(.2,.8,.2,1)`（Material），`--ease-spring: cubic-bezier(.34,1.56,.64,1)`（儿童交互用微回弹），`--dur-fast: 120ms / --dur-base: 220ms / --dur-slow: 480ms / --dur-ring: 900ms`。CTA 按钮 hover 用 `fast`，卡片展开用 `base`，进度环用 `slow + spring`。
- **优先级**：P1

## 2. 布局与壳层（+layout.svelte）

### 2.1 顶栏空洞、缺少导航语境
- **现状**：左上 logo + 文字 "ADHD-Screener"，右上健康芯片；中间大片留白。
- **问题**：(1) 应用名直接暴露"ADHD"英文缩写，儿童不懂，家长在公共场合（教室）打开有隐私不适；(2) 顶栏无面包屑 / 页面标题，导航感差；(3) 健康芯片的 "version" 数字对用户无意义。
- **建议**：
  - 对外显示名改为中文友好名（如"小专注"或"注意力小侦探"），副标题英文版保留在设置页。
  - 顶栏中间放当前页标题（"早筛 / 被试 / 训练"），或面包屑；左侧 logo 变纯图标。
  - 健康芯片简化：绿点=就绪 / 橙点=摄像头未连 / 红点=错误；不显示版本号，版本挪到"关于"页。
- **优先级**：P0

### 2.2 底部 tab 图标不够具象、层级不清
- **现状**：4 个 tab 都用 stroke-only 线性图标，active 状态只换颜色。
- **问题**：(1) 儿童识别 stroke 图标困难，尤其是 "rocket / gamepad / chart" 在 24px 下太抽象；(2) active 仅换色，缺少 affordance（胶囊背景 / 小圆点 / 图标跳动）；(3) 被试 tab 的"child"图标对儿童本人来说很奇怪（我是谁？"儿童"？）。
- **建议**：
  - 改为 duotone 图标（stroke + 半透明 fill），active 时 fill 升到 100%，inactive 时 fill 0%。参考 Phosphor duotone / Lucide 的 filled 变体。
  - active 态加胶囊底色：`background: var(--primary-50); border-radius: 20px; padding: 6px 14px`，图标 + 文字都在胶囊内，文字只在 active 显示（省空间 + 清晰）。
  - "被试"tab 语义更名："我的小伙伴"或直接是头像轮播；避免"被试"这个临床词被儿童看到。
- **优先级**：P0

### 2.3 首页即重定向，浪费入口
- **现状**：`/` 直接 `goto('/screening')`，显示"正在加载…"。
- **问题**：应用首开没有欢迎感；儿童反复打开只看到空文字，家长也没有信任锚点（"这个软件在做什么"）。
- **建议**：把 `/` 改成真正的"首页 / 仪表盘"，显示：大欢迎语（按儿童名字问候）+ 今日任务卡（"今天还没有早筛哦"）+ 上次结果摘要 + 两个 CTA（"开始早筛" "去训练"）。或者在没有被试时才重定向到 `/subjects/create`，有被试则到仪表盘。
- **优先级**：P1

## 3. 早筛开始页 (/screening)

### 3.1 hero 卡片插画太简陋（同心圆+十字）
- **现状**：SVG 是几个同心圆 + 四向短线，白色半透明堆在橙色渐变上。
- **问题**：与页面副标题"Sternberg 工作记忆"不呼应；给儿童的感觉像"靶心 / 瞄准"，略有压迫。家长看也觉得"简单了"。
- **建议**：换成一个友好插画 —— 例如小孩眼睛前面漂浮 3–5 个小光点（呼应范式里"记住圆点"），风格采用 Lottie 或静态 duotone SVG；或用一个正在集中注意力的小浣熊/小狐狸吉祥物（6–9 岁共识）。hero 背景用 `linear-gradient(135deg, #FF8C42, #E07030)` 再叠一层 `radial-gradient(circle at 80% 20%, rgba(255,209,102,.4), transparent 50%)` 做光晕。
- **优先级**：P0

### 3.2 "闯关流程"三步卡片视觉重量均等
- **现状**：3 张卡片视觉重量一样。
- **问题**：这 3 步中只有"步骤 2"是主任务（25 分钟），步骤 1 是准备，步骤 3 是结果。全部等权会让儿童以为"要做 3 次一样累的事"。
- **建议**：
  - 步骤 2 放大（更大的 badge、彩色条纹左边界 `border-left: 4px solid var(--primary)`、说明文字多一行）；1 和 3 缩小为次级卡片。
  - badge 用插画替代数字："1" 换成小眼睛图标、"2" 换成脑子/闪电、"3" 换成奖杯/星星；数字仍小字保留在角标。
- **优先级**：P1

### 3.3 被试选择用原生 `<select>`，体验割裂
- **现状**：`<select>` 原生下拉。
- **问题**：儿童点不中、不好看、macOS 原生下拉与品牌完全脱离；也看不到头像和性别。
- **建议**：改成横向滑动的头像卡片（类似 Netflix "谁在观看"）：每张卡片上是彩色 avatar 首字 + 名字，选中状态加 2px 品牌色描边 + 轻微放大 1.04。若只有 1 个被试则自动选中并隐藏滑动区。
- **优先级**：P0

### 3.4 "开始闯关"按钮缺少呼吸感与环境光
- **现状**：`.btn-primary` 实色橙 + 向上 1px 悬浮。
- **问题**：页面里这个按钮是最重要的 CTA，但视觉权重和普通按钮一样。
- **建议**：按钮 `background: linear-gradient(135deg, #FF8C42 0%, #E07030 100%)`；默认态加一圈 `box-shadow: 0 8px 24px -4px rgba(255,140,66,.45), 0 2px 4px rgba(255,140,66,.3)`；hover 时 shadow 扩散 + 内部叠一层 `radial-gradient(at 30% 30%, rgba(255,255,255,.25), transparent 60%)` 的高光。可选：按钮外一圈极慢的 pulse 光环（2s 循环，scale 1→1.04，opacity 1→0），吸引儿童。
- **优先级**：P1

## 4. 被试管理页 (/subjects)

### 4.1 空状态插画与成人被试场景不符
- **现状**：大头像 SVG（圆脸 + 橙色）做空态。
- **问题**：插画尺寸占屏幕 1/3，但下方"点击上方按钮"指引箭头缺失；家长第一次进来不知道"上方按钮"在哪。
- **建议**：插画缩小 1/3（`120px → 80px`），下方放一个 ghost 风格的大"+ 添加第一个小伙伴"卡片（`border: 2px dashed var(--primary-light); border-radius: var(--radius-lg); padding: var(--space-xl)`），点击整卡跳转。视觉上直接告诉用户"在这里点"。
- **优先级**：P1

### 4.2 被试卡片太密，缺少次级信息
- **现状**：一行 = 头像 + 名字 + id + 性别徽章。
- **问题**：(1) 没有"上次筛查时间 / 累计筛查次数 / 最近风险等级"这些家长/老师真正关心的信息；(2) 整卡点击后没有"进入详情"的视觉预期（没有 chevron 或 hover）。
- **建议**：
  - 卡片改为 2 列布局：左侧 avatar + 名字 + id（小字），右侧纵向堆叠 2 个 mini stat（"上次筛查：3 天前" / "累计：5 次"）+ 一个小 chevron。
  - avatar 背景色数组太艳（8 个满饱和色并排），改为每个颜色叠一层 `radial-gradient(circle at 30% 30%, rgba(255,255,255,.4), transparent 50%)` 模拟球面光；或者使用插画头像（动物头 / 水果头，6–9 岁喜欢）。
  - 卡片 hover 加 `transform: translateY(-2px); box-shadow: var(--shadow-md)`。
- **优先级**：P1

### 4.3 "添加被试"按钮在角落且词汇过于临床
- **现状**：右上角 `.btn-primary` "+ 添加被试"。
- **问题**：(1)"被试"是 research jargon，家长老师都会觉得冷漠，儿童完全不懂；(2) 按钮在 sticky 的 list 之上，滚动时会丢失。
- **建议**：词汇替换全局："被试"→"小伙伴 / 孩子 / 学生"（按场景切换，可做角色开关）。按钮升级为 FAB：右下角浮动圆形（`width: 56px; border-radius: 50%`），覆盖在底部 tab 之上 16px，儿童拇指热区。
- **优先级**：P0

## 5. 早筛进行中 (/screening/running)

### 5.1 "正在分析"页的进度环与 spinner 冗余
- **现状**：大环（240px，显示 progress %）+ 下方再放一个 32px 圆环 spinner + 标题 + 副标题 + stats chips，一共 5 个视觉焦点堆叠。
- **问题**：儿童看完任务后注意力已经很累，这个"分析中"页应该是"喘口气的奖励页"，现在却视觉最密集。
- **建议**：
  - 保留大环（让环自己做 spin 动画，stroke-dasharray 循环偏移），干掉下面的小 spinner。
  - 标题 + 副标题合并为一行 "分析中…"；stats 两个芯片保留但改小字号，横向均分。
  - 加一个小的鼓励语："做得好，马上就有报告啦" + 动物吉祥物小图（呼应其他页）。
- **优先级**：P1

### 5.2 stats chip 视觉过重（图标方块 44×44 + 粗边 + 大数字）
- **现状**：芯片里图标方块 44px 圆角 14，背景用品牌色 15% opacity。
- **问题**：芯片本身应该是"信息块"不是"操作块"，视觉太重抢了进度环的主角。
- **建议**：图标方块缩到 32px，去掉背景色（直接线性图标），`border: 1px solid var(--neutral-100)` 替换当前的 `#F0E0CC`；chip 内 label 12px 常规、value 18px semibold，label 在上 value 在下。
- **优先级**：P2

### 5.3 出错状态视觉过于直白（大红叉 × 在红色圆圈里）
- **现状**：红色 X 叉 + 红色标题"出错了"。
- **问题**：儿童看到会有挫败感，家长也会紧张（特别是医疗场景）。
- **建议**：图标换成"问号云朵"或"暂停"(`pause`) + 黄色/琥珀警示色（而非红色），标题改"哎呀，稍等一下"+ 副标题"这不是你的原因哦，我们再试一次"+ 主按钮"重新开始"，次按钮"先退出"。
- **优先级**：P1

## 6. 早筛结果页 (/screening/result)

> 注意：用户描述提到雷达图 + 疲劳时间线，但当前实现只有概率环 + feature importance 横条。以下按当前实现给出改进。

### 6.1 概率环 + 风险徽章组合在医疗语境下太"指控感"
- **现状**：大环显示 "72%" + 下方红/黄/绿徽章"高风险"。
- **问题**：(1) 把一个 ML 预测的概率值显示到个位百分比，会被家长误读为"精确诊断"；(2) 红色徽章对孩子有负面影响；(3) 顺序上先看到数字再看到文字解释，易先入为主。
- **建议**：
  - 环不显示百分比，改显示定性分段（如 4 段：稳定 / 关注 / 建议观察 / 建议就诊），环 stroke 分色段填充（像 Apple Watch activity ring），指针式箭头指向当前段。
  - 徽章语言去临床化："高风险"→"建议和专业医生聊聊"；"低风险"→"目前看起来很棒"。
  - 概率数字挪到家长说明卡内折叠展开，标注"这是模型估计，不是诊断"。
- **优先级**：P0

### 6.2 "给孩子的话"卡片内容千篇一律
- **现状**：无论结果都显示"你完成了所有的挑战！你的专注力表现非常棒，继续保持！"。
- **问题**：结果是"高风险"时，这句话变得讽刺；儿童能感受到"大人在糊弄我"。
- **建议**：根据 `risk_level` 给 3 套文案。共同点：肯定坚持完成的努力（这是确定的事实）而不是表现质量。例如："你坚持做完了所有闯关，这件事本身就很了不起！" + 一句关于下一步做什么的建议（一起做个小游戏 / 出去玩 5 分钟）。配图：一只举着奖牌的小动物（Lottie 静帧）。
- **优先级**：P0

### 6.3 "给家长的说明"卡片文字密集、无层级
- **现状**：3 段纯文字段落，术语密集（Sternberg / 随机森林 / 27 项指标）。
- **问题**：家长扫读困难，容易只看到数字不看方法学；免责声明藏在底部灰条里，临床/法律风险。
- **建议**：
  - 改为 "3 个要点" 格式（3 个带小图标的小卡）：① 这个测试基于什么（眼动 + 反应时）② 结果怎么解读（3 档说明）③ 下一步建议（结合 risk_level 显示不同内容）。
  - 免责声明置顶而非置底，用 info 蓝底 + `!` 图标，不用灰色（灰色=不重要，但这条很重要）。
- **优先级**：P0

### 6.4 Feature importance 横条图对家长无意义
- **现状**：前 10 个特征的中文友好名 + 百分比 + 橙/青横条。
- **问题**：家长看"瞳孔峰值负荷差异 12.3%" 完全不知道是啥；即便汉化了术语仍是术语。
- **建议**：改为"6 维注意力画像"雷达图（用户描述里提到的），6 个维度对应人类概念：专注力 / 反应速度 / 稳定性 / 抗干扰 / 工作记忆 / 视觉警觉；每维给一句口语化解读。原始特征重要性折叠在"专业模式"开关后面（给临床医生看）。
- **优先级**：P0

### 6.5 缺少下一步行动（dead end）
- **现状**：结果页底部就是特征条，没有 CTA。
- **问题**：家长读完不知道做什么；儿童读完没成就感（没有"再玩一次"按钮）。
- **建议**：页底 sticky 两个按钮：主"保存到报告档案"/ 次"再做一次" + 分享/打印（医疗场景有用）。结果页底部加一个"推荐的训练游戏"卡片 → 引流到 `/training`。
- **优先级**：P1

## 7. 校准 Canvas (CalibrationCanvas.svelte)

### 7.1 背景 `#37302B` 深棕，与整体暖色调不协调
- **现状**：Canvas 全屏深棕。
- **问题**：从主应用（`#FFF8F0`）突然跳到深棕，视觉断层大；对 6–9 岁儿童，长时间盯深色背景 + 亮点容易产生眼疲劳和光晕后像。
- **建议**：背景改为深蓝夜空渐变 `radial-gradient(ellipse at center, #2A3142 0%, #1C2030 100%)`（偏冷色反而减轻黄光屏幕的眼疲劳），并在远处撒几个极淡的"星星"（2–3px 白点 opacity 0.15），增加沉浸感又不分散注意力；或直接用暖色同系的 `#3B2E24 → #2B1810` 径向渐变。
- **优先级**：P1

### 7.2 校准目标点视觉太"技术"（同心圆 + 外圈脉动）
- **现状**：青绿圆盘 + 外圈 ring + 辉光 + 下方黄色 pill 标号。
- **问题**：对儿童来说像个靶心而不是"要注视的东西"；13 个点全一模一样也会无聊。
- **建议**：
  - 目标点改成"小星星 / 小萤火虫"：一个中心发亮的圆点 + 3–5 根向外的 sparkle 短线（旋转），颜色保留 `#4ECDC4`。
  - 13 个点可以用不同形状做成"点亮一个小星座"：第 1 点是小星星，点击后留下一条淡线连到第 2 点（形成连线动画），13 个全部完成后形成一个小星座图案闪烁一下 —— 儿童 intrinsic motivation 大幅提升。
  - 下方黄色 pill 的数字标号可去掉（儿童不需要知道是第几个，banner 里已有 1/13 提示）；或改成小图案（一朵云、一片树叶）。
- **优先级**：P0

### 7.3 顶/底切换的 banner 不稳定
- **现状**：banner 根据当前目标在上半屏还是下半屏切到对侧；文字"[1/13] 请注视圆点并点击"。
- **问题**：banner 每个点切换时横跳会分散儿童注意力；文字"请注视圆点并点击"对 6 岁偏难。
- **建议**：
  - banner 固定在顶部 20px（目标不会出现在最顶部 10%，所以不冲突），进度改成 1/13 小圆点串（已完成=填充 primary，未完成=空心），语言化操作提示永远不变："看着小星星 → 点一下它"。
  - 当前点序号改用一个友好说法："还差 12 颗星星啦"。
- **优先级**：P1

### 7.4 "平均误差: 23.4 像素"这种反馈儿童看不懂
- **现状**：校准完成页显示"平均误差: 23.4 像素"。
- **问题**：儿童完全不懂；家长也只是大概知道"越小越好"。
- **建议**：量化结果转定性："校准得很好！"（<30px） / "不错，可以开始了"（30–60） / "再校准一次会更好哦"（>60，给重试按钮）。像素值折叠在"专业信息"里（长按或开关）。
- **优先级**：P1

### 7.5 退出按钮半透明白在深棕上对比度不足
- **现状**：`background: rgba(255,255,255,.12); color: rgba(255,248,240,.7)`。
- **问题**：WCAG 对比度不达 3:1；家长想中止会找不到按钮。
- **建议**：退出按钮改成纯色背景（`background: rgba(255,255,255,.9); color: var(--text); backdrop-filter: blur(10px)`）+ 小"X"图标前置；对比度立即达标。不用担心分散注意力 —— 校准时目标点是主焦点，退出按钮在对角只在需要时被看见。
- **优先级**：P2（可及性）

## 8. 跨页面共性问题

### 8.1 loading 态全是"一个 spinner + 文字'加载中...'"
- **现状**：`subjects` / `result` / `start` 每页都一样。
- **问题**：(1) 空等待体验差；(2) 看不出要等多久。
- **建议**：统一改为 skeleton（骨架屏）：`subjects` 页显示 3 个灰色卡片骨架；`result` 页显示环 + 卡片的轮廓灰块。骨架用 `linear-gradient(90deg, #F0E8E0 0%, #FFF8F0 50%, #F0E8E0 100%); background-size: 200% 100%; animation: shimmer 1.5s infinite`。
- **优先级**：P1

### 8.2 error 态样式不一致 + 文案偏技术
- **现状**：不同页 `.error-state` / `.error-msg` / red text + "重试" 按钮各写各的；且文案是 `加载报告失败: ${e}` 这种带原始错误字符串。
- **问题**：(1) 用户看到 "TauriError: IpcInvokeError..."；(2) 样式不统一。
- **建议**：抽取 `<ErrorCard>` 组件（友好图标 + 口语化标题 + 原始错误折叠在"详情"内）。统一文案模式："哎呀，[动作] 遇到了点麻烦 —— [重试 按钮]"。技术细节只在开发模式显示。
- **优先级**：P1

### 8.3 页面切换无过渡
- **现状**：SvelteKit 默认无 page transition，跳转瞬间白屏 + 瞬间新页面。
- **问题**：儿童 UI 应用尤其需要视觉连续性；突变跳转让人觉得"卡了"。
- **建议**：`+layout.svelte` 给 `{@render children()}` 外套一个 `animate-fade-in`（已有 keyframe）+ 根据方向做 12px y-translate。更进一步：`main-content` 用 `view-transition-name`（新 CSS API）做 morph 过渡。
- **优先级**：P1

### 8.4 硬编码的 `height: calc(100vh - 60px)` 在顶栏改高时会碎
- **现状**：`.page-content { height: calc(100vh - 60px) }`。
- **问题**：60 是 top-bar 写死的，不是 token。
- **建议**：`--topbar-height: 60px` 变量化；或改用 flex（layout 已经是 flex 了，page-content 用 `flex: 1` + `min-height: 0` + `overflow-y: auto`，不要算高）。
- **优先级**：P2

## 9. 儿童 UI 特定问题

### 9.1 文字密度过高
- **现状**：所有页面都是"标题 + 段落 + 段落"结构，尤其结果页家长卡和早筛开始页的步骤描述。
- **问题**：6–9 岁儿童阅读广度每行最多 10–14 个汉字；当前段落每行 30+ 字。
- **建议**：
  - 给儿童看的内容：每行 ≤ 14 字（通过 `max-width: 22ch` 控制），字号 ≥ 18px，行高 1.7+。
  - 长段落全部换成项目符号或插画卡，避免超过 3 行纯文字。
  - 家长看的内容（如结果页 parent-card）可以保留段落，但字号 16px 即可。
- **优先级**：P0

### 9.2 缺少吉祥物 / 一致的视觉角色
- **现状**：SVG 图标零散，没有一个贯穿所有页面的"角色"。
- **问题**：儿童记忆 UI 靠角色而非文字；没有吉祥物 = 没有情感锚点。
- **建议**：设计一个简笔小动物（建议小狐狸 / 小浣熊 / 小猫头鹰 —— 注意力 / 机敏联想），在以下位置出现：hero 卡右下角、空态插画、结果页 congrats、错误页、训练页等待。单一角色 3–5 个姿态即可（开心 / 思考 / 加油 / 鼓掌 / 睡觉）。用 Lottie 或 SVG sprite。
- **优先级**：P1

### 9.3 缺少声音反馈
- **现状**：所有交互无声。
- **问题**：6–9 岁的任务闯关体验没有声音=没有奖励感；校准点击对不准也没有听觉提示。
- **建议**：加一套极简音效（各 50–100ms，音量克制）：成功 tap（清脆木琴 up）/ 校准完成（glockenspiel）/ 错误（柔和 buzz，不要尖锐）/ 每关通过（小号胜利短句）。必须提供静音开关（临床场景需要），默认教室/家庭模式都开，医院模式关。
- **优先级**：P1

## 10. 专业性 vs 亲和力平衡

### 10.1 需要"角色模式"开关
- **现状**：所有用户看到同一套 UI，包括临床医生看校准结果也会看到"小星星"。
- **问题**：在医院情境，如果医生/心理咨询师来演示，"小星星连线"显得不专业；在学校情境则刚刚好。
- **建议**：顶栏一个极小的角色切换（儿童模式 / 家长模式 / 专业模式），主要影响：
  - 语言严肃度（"小伙伴" vs "儿童" vs "被试"）
  - 动画密度（全部 vs 部分 vs 关闭装饰动画）
  - 数据呈现（友好化 vs 完整 vs 原始特征值）
  保存在本地设置，默认"儿童模式"。
- **优先级**：P1

### 10.2 视觉"童趣"与"医疗"比例建议 40 / 60
- **现状**：当前比例感觉约 70（童趣）/ 30（医疗）。
- **问题**：家长/老师首次打开会担心"这是个玩具 app 吗，能信任结果吗"。
- **建议**：
  - 保留童趣的地方：任务闯关（Canvas 内）、校准（小星星）、结果页的"给孩子的话"卡、吉祥物穿插。
  - 需要更克制的地方：顶栏、被试管理、结果页的"给家长说明"和"关键指标"、报告列表页。这些区域用更窄的色域（primary + neutral 为主，减少 tertiary 黄色的出现）、更严格的留白（卡片间距加大到 24px+）、字重克制（不要一堆 800 weight）。
  - 参考：Headspace Kids 首页 vs Headspace 成人首页 —— 同一个视觉语言，但色彩饱和度、插画密度、字重明显不同。
- **优先级**：P1

## 11. 动效与过渡 (P1)

### 11.1 进度环的 stroke-dashoffset 过渡过硬
- **现状**：`transition: stroke-dashoffset 0.5s ease`。
- **问题**：百分比跳变时 0.5s linear-ish 变化，不够"有机"。
- **建议**：改为 `transition: stroke-dashoffset var(--dur-slow) var(--ease-spring)`，或用 Svelte `tweened({ duration: 900, easing: cubicOut })` 驱动数值。每次 +1% 时环有一点"追上来"的弹性感。

### 11.2 卡片入场 fade-in 全部同时触发
- **现状**：多个 `.animate-fade-in` 同一帧开始。
- **问题**：缺少"有节奏地从上到下出现"的电影感。
- **建议**：给每个卡片加 `animation-delay: calc(var(--i) * 80ms)`，在模板里传 `--i` 序号。或用 Svelte `{#each ... as item, i}` + `transition:fly={{ y: 20, delay: i * 80, duration: 320 }}`。

### 11.3 按钮 active 下压太轻
- **现状**：`transform: translateY(0)` 从 -1px 回 0。
- **问题**：儿童喜欢"按得下去"的反馈。
- **建议**：active 态 `transform: translateY(1px) scale(.98); box-shadow: var(--shadow-sm)`，视觉上明显"陷下去"。

## 12. 可及性 (P2)

### 12.1 全局 `user-select: none` 过于激进
- **现状**：`body { user-select: none }`。
- **问题**：家长/医生想复制报告数字、session id 都复制不到。
- **建议**：默认允许选择；只在 Canvas 全屏任务页禁止（避免儿童误拖拽）。

### 12.2 focus 可见性全无
- **现状**：`button { outline: none }; input { outline: none }`。
- **问题**：键盘用户（临床场景可能用平板 + 外接键盘）完全看不到焦点。
- **建议**：统一 `:focus-visible { outline: 3px solid var(--primary); outline-offset: 2px; border-radius: inherit }`，只在键盘导航时显示（`:focus-visible` 不影响鼠标点击）。

### 12.3 色彩对比不足
- **现状**：`--text-muted #8B6F5C` 在 `--bg #FFF8F0` 上对比度约 4.3:1，对 12px 小字勉强达 AA。`disclaimer` 用 text-muted 当正文色对比度不够。
- **建议**：正文最低 `#6B4F3C`（≈5.5:1），小字用 `--text`；disclaimer 即使是次级信息，字号≥14 + 对比≥4.5。

### 12.4 缺少 ARIA 标签
- **现状**：`<svg>` 图标无 `role / aria-label`；进度环无 `role="progressbar"`。
- **建议**：装饰图标 `aria-hidden="true"`；功能图标加 `role="img" aria-label="..."`；进度环加 `role="progressbar" aria-valuenow aria-valuemin aria-valuemax aria-label="早筛进度"`。

## 附：最能立即提升质感的 1–2 个改动

### A. 把 Canvas 校准目标点从"靶心"改成"小星星连成星座"（7.2）
这是整个 app 最高光的交互点，每次早筛用户必然经过。当前"同心圆 + 黄 pill 标号"完全是研究工具美学；改成"一闪一闪的星星 + 点完留线 + 13 个点完成一个小星座闪烁一下"之后：(1) 儿童立刻获得 intrinsic motivation；(2) 家长和医生看到会觉得"这团队是用心在做儿童产品"；(3) 深色背景 + 星空 + 青绿发光星星的组合，本身就是非常上镜的视觉锤，可用作 app store 截图主素材。
成本：约 1 天（Canvas 绘制 + 连线 path 动画），无后端改动。

### B. 把结果页的 feature-importance 横条图改成 6 维雷达（6.4）
这是家长最常停留的界面。当前 10 根横条 + 术语（"瞳孔峰值负荷差异 12.3%"）对家长无意义，对孩子更糟糕。改成 6 维雷达（专注力 / 反应速度 / 稳定性 / 抗干扰 / 工作记忆 / 视觉警觉）后：(1) 家长 3 秒读懂孩子的画像；(2) 雷达图是公认的"专业感"可视化，立刻让 app 显得像临床工具；(3) 专业模式开关背后仍保留原始特征重要性给医生 —— 一鱼两吃。
成本：约半天（SVG 或 Chart.js radar），需要从后端 27 个特征映射到 6 维（简单线性加权即可先跑）。

这两个改动的共同特点：**视觉改动巨大、代码量小、覆盖所有用户的高频路径、同时提升"儿童亲和力"与"专业可信度"**。
