# 早筛体验优化清单

> 对象：6-9 岁儿童 + 家长陪同  
> 范式：Sternberg 视觉空间工作记忆任务（Wainstein 2017 / Rojas-Libano 2019）  
> **边界**：不改变范式计时、试次数、刺激结构、干扰类型分布；所有改动只作用于「范式外包装」。  
> 工作量估算：小 ≤ 半天 / 中 = 1-2 天 / 大 ≥ 3 天（单人 Svelte + Tauri）。

---

## 0. 总览（核心痛点）

1. **现有流程假设的是「合作型成人」**：3 秒 intro、F/J 按键、8 秒自动过 break、无练习试次、无故事包装——对 6-9 岁儿童来说，失败节点集中在「第一次看到指令」「第 80 试次起疲劳」「遇到情绪图片」三个位置。
2. **家长被完全排除在流程之外**：`start/+page.svelte` 不提示时长、不告知「要陪 30 分钟」、不解释结果页冲击，`running/+page.svelte` 进入范式后家长几乎看不到进度，结束后也没有家长专属的分层解读。
3. **异常场景静默失败**：ESC 退出丢全部数据、摄像头断开/人脸丢失均无反馈、校准失败无降级路径——这些对严肃临床数据是致命的。

## 1. 任务前引导 (P0)

### 1.1 儿童版指令卡 + 练习试次（缺失）
- **现状**：`SternbergCanvas.svelte:555-582` 的 `drawIntro` 只显示 3 秒，文字「F = 出现过 J = 没出现过」对 6-9 岁儿童完全不可懂；没有任何练习 trial。
- **问题**：第 1 个真实 trial 就是数据点；孩子在真正理解规则前已经错掉 5-10 个 trial，污染基线。
- **建议**：
  1. 在进入 `CalibrationCanvas` 之前插入一个独立「教学页」：左手食指贴 F 键、右手食指贴 J 键的**照片/图标**，配中文旁白「记住的话按左手 F，没记住按右手 J」。
  2. 加入 **4-6 个不计分的练习 trial**（范式外、不入 DB），每做完一个都显示「你做对了！」或「再来一个」，全部答对才能继续；两次答错自动弹出家长可读的提示。
  3. 练习 trial 的 load 固定为 1、干扰只用 type=6（blank），避免把未理解的规则难度和工作记忆难度混淆。
- **优先级**：**P0**
- **工作量**：**中**（复用 SternbergCanvas 渲染，加新 phase='practice'，不写数据入库）
- **参考**：NIH Toolbox Cognition Battery 的 Flanker 儿童版，练习 trial 强制通过 80% 才进主试。

### 1.2 家长知情同意 / 时长预告（缺失）
- **现状**：`start/+page.svelte:80-103` 只有「闯关流程」三步，**没有写出「约 30 分钟」「需要家长陪同不出声」「摄像头会打开」**。
- **问题**：家长中途接电话、离开、甚至不知道会用摄像头；孩子哭闹时家长也不知道该不该安抚。
- **建议**：在「开始闯关」按钮上方加一张折叠卡，明确写：
  - 预计时长：**约 30 分钟**（校准 1 分钟 + 8 关共 28 分钟 + 报告 1 分钟）
  - 摄像头：会录制**眼动（不录面部视频）**、数据**仅本地保存**
  - 家长角色：**坐旁边不说话、不提示答案**，孩子表示累时可在关卡之间示意休息
  - 一个「我已阅读并陪同」复选框，未勾选时「开始闯关」按钮禁用。
- **优先级**：**P0**
- **工作量**：**小**
- **参考**：MindBoost、PEAK 的成人陪同条款。

### 1.3 环境准备自检页（缺失）
- **现状**：直接进校准；没有亮度、距离、坐姿检查。
- **问题**：孩子躺着做、屏幕离 1.5 米、背光窗户——校准误差 >120 px 但没人发现。
- **建议**：在教学页和校准之间插入一个 **10 秒自检**：显示摄像头前置预览 + 一个绿色人形框，要求孩子「脸对准方框、坐直」；只有人脸连续 2 秒在框内才允许进入校准。Rust 后端已有 MediaPipe/人脸检测能力，前端只需订阅 `gaze_frame` 的 `valid` 字段。
- **优先级**：**P0**
- **工作量**：**中**

## 2. 校准体验 (P0)

### 2.1 13 点校准故事化（未实现）
- **现状**：`CalibrationCanvas.svelte:267-316` 是 13 个青色圆点 + 黄色编号药丸，`:188-192` 的 banner 文字是「请注视圆点并点击」。
- **问题**：13 次机械点击对 6-9 岁极其枯燥，后半段已经开始东张西望，校准数据质量下降。
- **建议**：保留圆点的视觉中心（科学要求精确位置不变），但换一套叙事皮肤：
  - 旁白改为**「帮小宇宙飞船找到 13 颗星星！还剩 X 颗」**（`currentPointIndex+1` 即可算）
  - 点击时播放一个**短的光点收集动画**（80-120ms，不延长流程）
  - 每 3-4 颗点完成时 banner 闪一下「太棒了！还有 Y 颗」
- **科学约束**：动画只能在点击**之后**、跳到下一点**之前**，不能在凝视期间加粒子/音效（会影响瞳孔）。
- **优先级**：**P0**
- **工作量**：**小**（只改 banner 文本 + 点击后 120ms 的收集动画）
- **参考**：Zwigato 儿童注意力测评、iPad 版 NIH Toolbox 用「找宝藏」包装 Flanker。

### 2.2 点击容错与重试（部分实现）
- **现状**：`CalibrationCanvas.svelte:48, 121-123` 使用 `CLICK_RADIUS = 40px`，点偏了**静默忽略**，没有任何反馈。
- **问题**：孩子点了三次没反应，会以为系统坏了、开始乱点，或直接喊家长。
- **建议**：
  1. 点击偏离时显示一个**短暂的灰色涟漪** + 「再靠近一点试试」半透明提示（600ms）。
  2. 同一点连续失败 3 次 → 放大 CLICK_RADIUS 到 70px、并把圆点放大 1.3 倍再试一次；仍失败则记录该点为「跳过」，但记一条 warning 到会话元数据。
- **优先级**：**P0**
- **工作量**：**小**

### 2.3 校准失败的降级路径（缺失）
- **现状**：`handleClick` 在 13 点完成后调用 `finish_calibration`（`:140`），得到 `meanError`；但**无论误差多大都继续**（只显示数字）。5 点验证也不检查是否通过。
- **问题**：误差 150+ px 时后续所有瞳孔/注视特征都污染，但流程照走；对家长来说 30 分钟白做。
- **建议**：
  - `meanError > 阈值`（建议 60 CSS px）→ 弹「校准不太准，我们再来一次？」选项：**重试** / **降级继续**（会在最终报告里标注「眼动特征可靠性低」并只用行为特征打分）/ **放弃本次**。
  - 5 点验证期间实时计算注视偏差，若任一点偏差 > 80 px → 同样进入上面的三选一。
  - 校准元数据（mean_error、每个点的落点偏差）**随会话一起保存**，报告页可查。
- **优先级**：**P0**
- **工作量**：**中**（需在 Rust 端暴露 validation error，前端加弹窗+路由）

## 3. 任务进行中动机维持 (P0)

### 3.1 即时反馈已有但分级过简（部分实现）
- **现状**：`SternbergCanvas.svelte:697-761` `drawFeedback` 只有二元「太棒了 / 没关系」，持续 1.5 秒；没有连对奖励。
- **问题**：孩子连对 8 个，得到的反馈和随便答对一个一样，边际激励快速衰减。
- **建议**（**不修改 FEEDBACK_MS**）：
  - 在同一个 1.5s 反馈帧里，根据**最近 5 题的连对计数**分级：
    - 连对 1-2：原样
    - 连对 3-4：加一颗金星 + 文字「3 连击！」
    - 连对 5+：加两颗星 + 「高手！」
    - 连错 3+：文字改成「慢慢来，看清楚再按」（不加负面图形）
  - 所有叠加元素只用**静态图形**，不加音效、不加动画（避免污染下一 trial 的 fixation）。
- **优先级**：**P0**
- **工作量**：**小**
- **参考**：NIH Toolbox Flanker 儿童版的「streak 机制」。

### 3.2 Block 间 8 秒自动继续对儿童太短（已实现但需调整）
- **现状**：`SternbergCanvas.svelte:352-357` 如果不按空格，**8 秒后自动进入下一 block**；`:807` 的提示是「按空格键继续」。
- **问题**：孩子可能在喝水/揉眼睛，8 秒自动继续导致 block 开头前几题全错；而且「空格键」对儿童不直观。
- **建议**：
  - 默认休息**延长到 20-30 秒**（范式没有硬性规定 8s），同时提供**显式的大按钮**「准备好了，继续 →」（鼠标点击或空格任选）。
  - 休息页加一个 **圆形倒计时 20 秒**「休息时间还有 XX 秒」+ 一个**进度地图**（8 个格子，亮起 N 个）。
  - 第 4 关（正好一半）和第 6 关休息时间**自动延长到 45 秒**且弹出「起来走两步吧」文字，与 3.3 的疲劳监控联动。
- **优先级**：**P0**
- **工作量**：**小**

### 3.3 疲劳 / 中场大休息（缺失）
- **现状**：无任何疲劳监测反馈；160 试次从头到尾节奏一致。
- **问题**：第 80 试次（block 4/8 结束）是儿童注意力崩溃的经典位置，但当前没有任何干预。
- **建议**：
  - 在 block 4 结束的 break 插入「**中场休息**」专属界面：插画 + 「已经完成一半啦！站起来伸伸懒腰、喝口水，大约 60 秒后继续」+ **60 秒强制休息**（不能提前跳过）。
  - 若后端已有实时瞳孔/注视方差指标（见 `/features/…`），在 break 页显示「累了吗？」二选一按钮，选「累了」则休息延长到 90 秒。**重要：此选择不影响范式时序，仅作用于 break 时长 + 元数据标签**。
- **优先级**：**P1**（实现门槛取决于是否已有实时疲劳信号）
- **工作量**：**中**

### 3.4 试次计数可视化（缺失）
- **现状**：`drawBreak` 只显示 block 进度（8 个里第几个）；trial 内不显示剩余题数。
- **问题**：对儿童是好事（不制造焦虑），但家长没有任何外部视角知道还有多久。
- **建议**：trial 进行中**屏幕上保持纯净**（不改），但在**家长视角**（见 §7）单独显示 `trial N/160`、`block X/8`、`预计剩余 YY 分钟`。
- **优先级**：**P0**
- **工作量**：**小**（随 §7 一起做）

### 3.5 break 页鼓励语过于笼统（部分实现）
- **现状**：`SternbergCanvas.svelte:104-113` 有 8 条 `breakMessages`，按 block 顺序轮流。
- **问题**：内容和孩子表现无关——连错 10 题仍然说「你做得非常棒」会显得假；连对的孩子又没得到个性化肯定。
- **建议**：根据上一个 block 的准确率动态选择：
  - acc ≥ 0.85：「你太厉害了，这关全对快要满分啦」
  - 0.65 ≤ acc < 0.85：「节奏很稳，继续保持」
  - acc < 0.65：「这关有点难，下一关我们试试慢一点按」
- **优先级**：**P1**
- **工作量**：**小**

## 4. 疲劳监控与实时反馈 (P1)

### 4.1 摄像头人脸丢失 5 秒提示（缺失）
- **现状**：`SternbergCanvas.svelte:865-868` 订阅 `gaze_frame`，但 `valid=false` 时只是不记录数据，**不通知用户**。
- **问题**：孩子偏头、趴桌上、走开——屏幕继续跑 trial，这段时间数据全丢但没人知道。
- **建议**：
  - 累计 `valid=false` 超过 **5 秒**（非严格连续，允许 200ms 毛刺）→ 在屏幕右上角出现一个**小橙色提示条**「请坐正看屏幕」+ 一只眼睛的图标（**不暂停任务**，避免破坏范式；trial 正常进行，只在数据里打 `face_lost=true` 标签）。
  - 累计 > 15 秒 → 当前 trial 结束后（下一个 fixation 前）**自动插入一个 short break 页**「我们休息一下～坐好继续」，家长按继续才跑下一 trial。**这打破了连续性但避免了数据崩塌**，需在知情同意里说明。
- **优先级**：**P1**
- **工作量**：**中**

### 4.2 瞳孔/注视异常实时标签（可选）
- **现状**：所有指标都是事后从 trial series 算出来的。
- **建议**：在 Rust 端维护一个滑窗瞳孔 std / 注视路径长度，超过基线 2σ 持续 3 trials → 在 session metadata 打「mid-task fatigue」标签，在报告页特征解读中纳入。**对儿童侧没有可见变化**。
- **优先级**：**P2**
- **工作量**：**中**

## 5. 干扰图片情绪处理 (P1)

### 5.1 情绪图片内容分级筛查（缺失）
- **现状**：`SternbergCanvas.svelte:127-145` 从 `/images/emotional/1-20.jpg` 加载，**没有对儿童适宜性的二次筛查**。经典范式常用 IAPS / NAPS 库，其中部分图片（如伤口、蛇、愤怒面孔）对 6-9 岁儿童**过度刺激**。
- **问题**：孩子看到惊吓图片会哭、笑场、或在后续 trial 心悸——500ms 干扰图片虽然短，但对儿童大脑足够产生残留情绪。
- **建议**：
  1. 建立**儿童版图片白名单**：只使用 NAPS-CH / GAPED 的「mildly negative」分位（不含伤害、血、蛇、虫）和 `happy/neutral` 面孔；所有图片经 1-2 名有儿童心理学背景的人员二次审阅。
  2. 在 `emotional/` 目录加一个 `manifest.json`，字段包含 `valence`、`arousal`、`child_safe=true/false`，加载时 filter。
  3. 报告里记录本次实际用到的图片 ID（可追溯）。
- **优先级**：**P1**（伦理+体验）
- **工作量**：**中**（主要是内容侧筛选，代码改动小）
- **参考**：NAPS-CH（Nencki Affective Picture System - Children）、GAPED 儿童分级。

### 5.2 完成后情绪安抚（缺失）
- **现状**：`drawComplete` 直接跳「正在生成报告」。
- **问题**：任务结束时孩子刚刚看过 40 张左右情绪图片，直接进「分析注意力」会把紧张情绪带到家长看报告时。
- **建议**：在 `drawComplete` 之后、goto result 之前插入一个 **8-10 秒的「平静收尾页」**：暖色背景、慢速呼吸圆环（4 秒吸 4 秒呼）、文字「闯关结束啦，深呼吸三次～」+「完成奖状」（见 §6.1）。
- **优先级**：**P1**
- **工作量**：**小**

## 6. 任务完成后 (P0)

### 6.1 儿童完成奖状（缺失）
- **现状**：`SternbergCanvas.svelte:810-842` `drawComplete` 只显示勾号+「所有关卡完成」+「正在生成报告」。
- **问题**：30 分钟的付出没有仪式感，孩子转头就忘；下次再让他做这个任务会抵触。
- **建议**：
  - 独立的**完成奖状页**（全屏、竖排证书式）：孩子名字 + 日期 + 一个 SVG 徽章（依据 block 4 是否坚持下来、总准确率分金/银/铜）。
  - 按钮「保存我的奖状」→ 把 SVG 转 PNG 存到本地 `~/Documents/ADHD-Screening/certificates/`。
  - 徽章绝不提「ADHD / 风险 / 诊断」字样；**不写任何评估结论**，只奖励「完成」这个行为。
- **优先级**：**P0**
- **工作量**：**中**

### 6.2 儿童/家长分界面（缺失）
- **现状**：`result/+page.svelte` 是一个技术性页面（26 个特征中文名全列、风险等级）——孩子和家长**看同一页**。
- **问题**：孩子看到「中风险 / 高风险」会直接受伤；家长看到 27 个特征名也看不懂。
- **建议**：
  - 结束立即进入 **儿童页**：只有奖状 + 「叫爸爸妈妈过来一起看」按钮。
  - 点击后要求**输入一个 4 位家长 PIN**（在 start 页让家长设一次，存在 session）→ 验证后才进入 **家长报告页**。
  - 家长报告页分两屏：第 1 屏「一张图总结」（风险等级 + 3 条最重要的行为观察），第 2 屏「完整特征解读」（当前的 27 项）。
- **优先级**：**P0**
- **工作量**：**中**

### 6.3 高风险结果的缓冲措辞（现状刺激）
- **现状**：`result/+page.svelte:73-80` 直接显示「低风险 / 中风险 / 高风险」三色。
- **问题**：对未经辅导的家长，「高风险」三个字可能引发焦虑、否认、或反而过度医疗化。
- **建议**：
  - 文案改为**非诊断化描述**：「建议进一步观察 / 建议与老师沟通 / 建议预约专业评估」，保留原始 risk level 在可折叠的「技术指标」区。
  - 高风险结果页强制显示一段 60-100 字的解读：「这个结果不是诊断。它表示 XX 维度与同龄样本有偏差，下一步建议的是……（专业机构线索）」。
  - 提供「打印 PDF 带去就诊」按钮。
- **优先级**：**P0**
- **工作量**：**小-中**

## 7. 家长视角 (P1)

### 7.1 家长旁观面板（缺失）
- **现状**：任务进行时全屏都是孩子的视图，家长只能看个大概；`running/+page.svelte` 在 running phase 直接把画布铺满。
- **问题**：家长不知道进度、不知道孩子是不是在认真做、不知道还要多久。
- **建议**：在**副屏** / **缩小窗口** 提供一个「家长视角」：顶部窄条（~60 px 高）显示
  - `Trial 47/160 · Block 3/8`
  - 一个水平进度条
  - 预计剩余 `18 分钟`
  - 小图标显示「人脸 √ / 眼动 √ / 按键 √」
  - 一个「安静暂停」按钮（仅紧急使用，会标记数据段）
  
  此条可由家长按 `Alt+P` 快捷键显示/隐藏（默认隐藏，不干扰孩子）。
- **优先级**：**P1**
- **工作量**：**中**

### 7.2 开始前家长操作清单（缺失）
- **现状**：start 页没有告诉家长「你在这 30 分钟要做什么」。
- **建议**：在 §1.2 的同意卡内嵌 3 条：
  - 孩子做题时**不要提示答案、不说话**
  - 孩子走神时**可以轻拍桌子一次**，但不要指屏幕
  - 关卡之间（看到「第 X 关完成」页）可以和孩子说「继续加油」
- **优先级**：**P1**
- **工作量**：**小**

## 8. 异常场景 (P0)

### 8.1 ESC 中途退出的数据保留（不明确）
- **现状**：`CalibrationCanvas.svelte:103-107` 的 ESC 走 `onCancel` → `goto('/screening')`；`SternbergCanvas.svelte` 的 key handler（`:424-438`）**不处理 ESC**；且 `handleAllTrialsDone` 只在 160 trials 全完时才调用 `finish_screening`。
- **问题**：孩子中途崩溃按 ESC → 已经做的 N 个 trial 的数据**全部丢失**。对严肃临床任务是巨大浪费（30 分钟 * 80% 儿童 = 大量废数据）。
- **建议**：
  1. 每个 trial 完成时 `submit_trial_result`（`:394`）已经在做，数据已**增量落盘**——确认 Rust 端是否真的立即持久化；若否，立即改成写入。
  2. 添加「紧急退出」按钮（不是 ESC，要求家长同时按 `Alt+Shift+Q`，避免孩子误触），触发后：
     - 弹窗「已完成 N/160 题，是否保留已做的数据生成部分报告？」
     - 保留 → 调用 `finish_screening_partial(session_id, completed_trials)`，生成一份「部分数据报告」，明确标注样本量不足、结论仅供参考。
     - 放弃 → 删除本 session。
  3. 孩子按 ESC 时显示一个**家长确认遮罩**（而非直接退出），家长按「继续」回到任务、按「真的退出」走上面的流程。
- **优先级**：**P0**
- **工作量**：**中**（需 Rust 端新增 `finish_screening_partial`）

### 8.2 摄像头断开中途恢复（缺失）
- **现状**：无摄像头错误处理；`gaze_frame` 事件停了就停了。
- **问题**：USB 松动 / 隐私遮挡滑块 → 后续所有眼动特征都是 0，范式还在跑。
- **建议**：
  - Rust 端暴露 `camera_status` 事件（连接/断开/人脸丢失）。
  - 前端 `SternbergCanvas` 订阅；断开时：
    - 当前 trial 结束后（不破坏正在进行的 trial）进入一个专属暂停页「摄像头连接断开了，请检查 USB 或隐私滑块」。
    - 恢复后**强制重做校准**（5 点快速验证，不必重做 13 点），然后从下一 trial 继续。
    - 中断元数据写入 session。
- **优先级**：**P0**
- **工作量**：**大**（涉及 Rust + 前端 + 范式续跑逻辑）

### 8.3 人脸丢失 > 5 秒提示（见 §4.1）
见 §4.1。此处补一条：提示条显示时，当前 trial **不打 `face_lost` 标签**（可能是短暂毛刺），**下一个 trial 才开始污染标记**；避免一次眨眼把整个 trial 判废。

### 8.4 系统休眠 / 屏保（缺失）
- **现状**：30 分钟流程中，macOS 默认 10 分钟进屏保。
- **建议**：screening phase 期间通过 Tauri `window.request_user_attention` + 调用 macOS `caffeinate`（Rust 端 `std::process::Command`）阻止休眠，结束后释放。
- **优先级**：**P0**
- **工作量**：**小**

## 报告：Top 提升

### Top 5 最能提升儿童完成率的改进

1. **§1.1 练习 trial + 儿童版按键图示** —— 解决「第 1 个 trial 就是污染数据」的根本问题，同时让孩子感到「我会玩」。
2. **§3.2 Block 间休息延长 + 显式按钮 + 中场大休息** —— 8 秒自动继续 → 20/60 秒可控休息，直接降低 block 4 后的崩溃率。
3. **§6.1 完成奖状** —— 给 30 分钟一个心理收尾，决定孩子**愿不愿意再做一次**（随访/复评的关键）。
4. **§2.1 + §2.2 校准故事化 + 点击失败反馈** —— 校准是孩子接触系统的前 1 分钟，枯燥+无反馈的设计会让孩子从一开始就抵触。
5. **§3.1 连对分级反馈** —— 在不改范式计时的前提下，把 160 次反馈从「二元」升级到「有成长感」，是整个 30 分钟内唯一的内在激励来源。

### Top 3 最能减少家长焦虑的改进

1. **§6.2 + §6.3 儿童/家长分界面 + 非诊断化措辞** —— 避免「高风险」三个字直接砸向家长和孩子，同时给出可执行的下一步。
2. **§1.2 知情同意 + 时长预告** —— 家长提前知道要陪 30 分钟、摄像头在做什么，焦虑来源的 80% 来自「不知道」。
3. **§7.1 家长旁观面板** —— 「还剩 18 分钟、人脸√、按键√」三行信息即可让陪同家长从焦虑的旁观者变成知情的协作者。

## 附：明确不做的事（保护范式刚性）

- 不改变 `FIXATION_MS / ENCODING_DOT_MS / ENCODING_GAP_MS / MAINTENANCE_MS / DISTRACTOR_MS / PROBE_MS / FEEDBACK_MS` 任何一个。
- 不加音效（会干扰瞳孔；除非未来做带听觉版本的范式变体）。
- 不跳过、不重排、不新增 trial（practice trial 独立于 160 trials 之外）。
- 不在 encoding / maintenance / distractor / probe 期间做任何 UI 叠加（包括进度条、提示条）。
- 不基于实时准确率动态调 load / 难度（会破坏特征的跨被试可比性）。
